      # Docker入门

      本文意在帮助读者初步了解docker,并快速部署自己的应用

      # 一、docker简介

      docker 主要能解决 环境一致性、系统资源分配、服务节点扩容 问题，主要体现为

      集装箱：保证所有的运行环境、配置文件等齐全，确保程序运营不会缺东西 标准化：运输、存储、API接口的一直性，均为标准docker命令 隔离性：运用LXC(轻量级的容器虚拟化技术).最大效率的隔离了进程和资源

      # 二、docker结构

      docker运行程序的过程就是去仓库把镜像拉到本地,然后用一条命令把镜像运行起来变成容器.

      build:构建,就是构建镜像. ship:运输,运输镜像,从仓库和主机运输. run:运行的镜像就是一个容器. build,ship,run和镜像,仓库,容器是一一对应的.
